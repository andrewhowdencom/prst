# Getting Started

This tutorial walks you through installing `prst`, creating your first configuration file, and seeing a custom prompt in Bash.

## What you'll learn

- How to install `prst`.
- Where to put your configuration file.
- How to configure a basic colored prompt.
- How to reload your shell so the changes take effect.

## Prerequisites

- Bash (any recent version).
- Go installed (to build from source), or a pre-built binary.

## Step 1: Install prst

### Build from source

```bash
git clone https://github.com/andrewhowdencom/prst.git
cd prst
task build
```

This produces a `./prst` binary in the project root. Move it somewhere on your `$PATH`, for example:

```bash
mv ./prst ~/.local/bin/
```

Verify the installation:

```bash
prst version
```

You should see `v0.0.0-dev`.

## Step 2: Create the configuration directory

`prst` reads its configuration from `$XDG_CONFIG_HOME/prst/config.yaml` (usually `~/.config/prst/config.yaml`). Create the directory if it doesn't exist:

```bash
mkdir -p ~/.config/prst
```

## Step 3: Write your first config

Create `~/.config/prst/config.yaml` with the following content:

```yaml
ps1:
  segments:
    - type: user        color: green
    - type: literal     text: "@"
    - type: host         color: cyan
    - type: literal     text: ":"
    - type: cwd          color: blue
    - type: literal     text: " "
    - type: prompt_char
```

This defines a prompt that looks like:

```
user@host:/current/path$
```

Each segment is colored, and `prst` automatically wraps color codes in Bash non-printing byte markers so cursor positioning stays correct.

## Step 4: Hook prst into Bash

Add the following line to your `~/.bashrc`:

```bash
PS1='$(prst 1)'
```

The single quotes are **important** — they prevent Bash from evaluating `prst` once when `.bashrc` is sourced. Instead, Bash evaluates `prst` fresh before every prompt.

## Step 5: Reload your shell

Either open a new terminal, or run:

```bash
source ~/.bashrc
```

Your prompt should now be generated by `prst`. Try `cd`-ing into different directories to see the path update automatically.
