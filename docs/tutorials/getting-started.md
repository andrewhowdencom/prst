# Getting Started

This tutorial walks you through installing `prst`, creating your first configuration file, and seeing a custom prompt in your shell.

## What you'll learn

- How to install `prst`.
- Where to put your configuration file.
- How to configure a basic colored prompt.
- How to reload your shell so the changes take effect.

## Prerequisites

- Bash or Zsh (any recent version).
- Go installed (to build from source), or a pre-built binary.

## Step 1: Install prst

### Build from source

```bash
git clone https://github.com/andrewhowdencom/prst.git
cd prst
task build
```

This produces a `./prst` binary in the project root. Move it somewhere on your `$PATH`, for example:

```bash
mv ./prst ~/.local/bin/
```

Verify the installation:

```bash
prst version
```

You should see `v0.0.0-dev`.

## Step 2: Create the configuration directory

`prst` reads its configuration from `$XDG_CONFIG_HOME/prst/config.yaml` (usually `~/.config/prst/config.yaml`). Create the directory if it doesn't exist:

```bash
mkdir -p ~/.config/prst
```

## Step 3: Write your first config

Create `~/.config/prst/config.yaml` with the following content:

```yaml
ps1:
  segments:
    - type: user        color: green
    - type: literal     text: "@"
    - type: host         color: cyan
    - type: literal     text: ":"
    - type: cwd          color: blue
    - type: literal     text: " "
    - type: prompt
```

This defines a prompt that looks like:

```
user@host:/current/path$
```

Each segment is colored. Color codes are handled by your shell's initialization wrapper so cursor positioning stays correct.

## Step 4: Hook prst into your shell

### Automatic installation (recommended)

Run the install command. It auto-detects your shell and appends the correct initialization block to your rc file:

```bash
prst install 1
```

For zsh, or to install multiple prompt variables:

```bash
prst install --shell zsh 1 2
```

The command is idempotent — running it again updates the block rather than duplicating it.

### Manual installation

If you prefer to edit your rc file by hand, add the output of `prst init`:

**Bash** — add to `~/.bashrc`:
```bash
eval "$(prst init bash 1)"
```

**Zsh** — add to `~/.zshrc`:
```zsh
eval "$(prst init zsh 1)"
```

The `init` script outputs wrapper functions that invoke `prst` before every prompt and wrap ANSI sequences in the appropriate non-printing markers for your shell.

## Step 5: Reload your shell

Either open a new terminal, or run:

```bash
# Bash
source ~/.bashrc

# Zsh
source ~/.zshrc
```

Your prompt should now be generated by `prst`. Try `cd`-ing into different directories to see the path update automatically.
